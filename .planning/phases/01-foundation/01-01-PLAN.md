  <files>

    frontend/src/vite-env.d.ts
  </files>
  <action>
    Create the frontend directory and scaffold a Vite + React + TypeScript project:

    1. Run `npm create vite@latest frontend -- --template react-ts` from project root
    2. Navigate to frontend and install with pnpm: `cd frontend && pnpm install`
    3. Install core dependencies:
       ```bash
       pnpm add @tanstack/react-query @tanstack/react-query-devtools
       pnpm add lucide-react clsx tailwind-merge
       pnpm add tailwindcss @tailwindcss/vite
       pnpm add -D @types/node
       ```

    4. Update vite.config.ts to include Tailwind plugin and path aliases:
       ```typescript
       import path from "path"
       import tailwindcss from "@tailwindcss/vite"
       import react from "@vitejs/plugin-react"
       import { defineConfig } from "vite"

       export default defineConfig({
         plugins: [react(), tailwindcss()],
         resolve: {
           alias: {
             "@": path.resolve(__dirname, "./src"),
           },
         },
       })
       ```

    5. Update tsconfig.json to enable strict mode and path aliases:
       - Set "strict": true in compilerOptions
       - Add baseUrl: "." and paths: { "@/*": ["./src/*"] }

    6. Update index.html title to "RAG Dashboard"

    7. Create a minimal App.tsx that just renders a heading "RAG Dashboard" for now
  </action>
  <verify>
    Run `cd frontend && pnpm dev` - app should start at http://localhost:5173 with "RAG Dashboard" heading visible.
    Run `cd frontend && pnpm tsc --noEmit` - should pass with no type errors.
  </verify>
  <done>
    Vite dev server runs successfully, TypeScript compiles without errors, and browser shows the React app.
  </done>
</task>

  <action>
    Initialize shadcn/ui and configure Tailwind v4 CSS:

    1. Run shadcn init from frontend directory:
       ```bash
       cd frontend && pnpm dlx shadcn@latest init
       ```
       When prompted:
       - Style: Default
       - Base color: Neutral (or Slate for a more Linear-like feel)
       - CSS variables: Yes
       - Tailwind CSS version: v4
       - Global CSS file: src/index.css
       - CSS variables for theming: Yes
       - Path aliases: @/components and @/lib (use existing from tsconfig)

    2. The init will create/update src/index.css with Tailwind v4 CSS-first configuration including:
       - @import "tailwindcss"
       - CSS variables for light theme (:root)
       - CSS variables for dark theme (.dark)
       - @theme inline directive mapping CSS vars to Tailwind

    3. Verify src/lib/utils.ts was created with the cn() helper:
       ```typescript
       import { clsx, type ClassValue } from "clsx"
       import { twMerge } from "tailwind-merge"

       export function cn(...inputs: ClassValue[]) {
         return twMerge(clsx(inputs))
       }
       ```

    4. Ensure index.css includes sidebar CSS variables (add if missing):
       ```css
       :root {
         --sidebar-background: hsl(0 0% 100%);
         --sidebar-foreground: hsl(222.2 84% 4.9%);
         /* ... other sidebar vars from research */
       }
       .dark {
         --sidebar-background: hsl(222.2 84% 4.9%);
         --sidebar-foreground: hsl(210 40% 98%);
         /* ... other sidebar vars from research */
       }
       ```

    5. Update main.tsx to import index.css if not already imported
  </action>
  <verify>
    Run `cd frontend && pnpm dev` - app should still run without CSS errors.
    Check that src/lib/utils.ts exists and exports cn function.
    Check that components.json exists with proper path aliases configured.
  </verify>
  <done>
    shadcn/ui is initialized, components.json exists, utils.ts has cn() helper, and index.css has Tailwind v4 configuration with theme CSS variables.
  </done>
